# tts-server

SkyRL-backed tinker API server for `tts-tokens-that-suffice`, packaged as a
self-contained ROCm (AMD GPU) Apptainer image.

Runs SkyRL's `skyrl.tinker.api` directly. (Earlier a `tts.tinker.api` wrapper
added the `/api/v1/client/config` endpoint; the pinned SkyRL now implements that
natively, so the wrapper was dropped.)

## Stack

Everything is version-aligned to SkyRL's own pins, so the image needs **no
source patches or runtime shims**:

| Component | Version | Notes |
|-----------|---------|-------|
| base image | `rocm/pytorch:rocm7.2.4_ubuntu24.04_py3.12` | venv at `/opt/venv` |
| torch | `2.11.0+rocm7.2` | official PyTorch ROCm wheel |
| vllm | `0.20.2` | built from source (gfx90a=MI250, gfx942=MI300X) |
| ray | `2.51.1` | |
| flash-attn | `2.8.3` | Triton-AMD backend (pure-Python) |
| SkyRL | pinned commit | cloned during the build (not a submodule) |

Runs on hosts with a ROCm 6.16.x+ amdgpu driver (ROCm 7.2 userspace is
forward-compatible).

## Build the image (one-time)

```bash
bash server/scripts/build.sh
```

Produces `../tts-server.sif`. The build:
1. Pulls the `rocm/pytorch` base image (once).
2. Installs torch 2.11 (ROCm), ray, the ROCm-compatible SkyRL deps, flash-attn,
   and `amdsmi` into the base venv `/opt/venv`.
3. Clones SkyRL at the pinned `SKYRL_COMMIT` and `pip install -e`'s it.
4. Builds vllm 0.20.2 from source for ROCm.

Re-run only after bumping `SKYRL_COMMIT` / vllm / torch in `build.sh`.

## Run the server

```bash
# SFT / supervised learning (default)
bash server/scripts/run.sh

# RL mode (enables vLLM inference engines)
bash server/scripts/run.sh rl
```

`run.sh` launches `tts-server.sif` with `--rocm`, binds `src/`, the HF cache,
checkpoints, and Ray's tmp dir, sets the ROCm/Ray GPU-visibility env, and runs:

```
uv run --no-sync --extra tinker --extra fsdp -m skyrl.tinker.api \
    --base-model 'Qwen/Qwen3-8B' --backend fsdp --port 9123 \
    --checkpoints-base /checkpoints --backend-config <config/{sft,rl}.json>
```

## Structure

```
server/
├── scripts/
│   ├── build.sh         builds ../tts-server.sif (run once)
│   ├── run.sh           launches the container server [sft|rl]
│   └── tts-server.def   generated by build.sh
├── config/
│   ├── sft.json         backend config for supervised learning
│   └── rl.json          backend config for RL (enables vLLM engines)
└── checkpoints/         checkpoint output (created on first run)
```

## Key env / GPU notes (set by `run.sh`)

- `ROCM_PATH=/opt/rocm` — the base image's default points at a nonexistent
  `/opt/rocm-7.2.0`; without this Ray's pyamdsmi finds no GPUs.
- `RAY_EXPERIMENTAL_NOSET_ROCR/HIP_VISIBLE_DEVICES=1` and **no** global
  `HIP_VISIBLE_DEVICES` — let Ray assign one GPU per actor via
  `CUDA_VISIBLE_DEVICES`; forcing HIP to all-8 conflicts with that.
- `UV_PROJECT_ENVIRONMENT=/opt/venv` — use the image's venv directly.

## Changing the model

Edit `--base-model` in `scripts/run.sh`.
